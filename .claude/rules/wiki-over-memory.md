# Wiki Over Memory

When the user asks to save, document, or record knowledge, save to the **wiki** or update a **skill**, not memory. The wiki and skills are shared across all agents, projects, and computers. Memory is personal and invisible to others.

MUST NOT use memory for conventions, patterns, decisions, or learnings that other agents or projects need. If it's a behavioral guardrail, update the relevant skill. If it's project knowledge, update the wiki. Memory is only for personal preferences.

**Reading priority:** when memory conflicts with the wiki, a rule, CLAUDE.md, or a skill, the versioned source wins. YOU MUST NOT use memory to override a MUST from a versioned source. Remove the stale memory and report the conflict.

Load Skill("wiki") for full conventions on what belongs where and how to handle conflicts.
